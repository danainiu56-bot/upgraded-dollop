# 产品到营销系统

> Mason 的产品 + 原型工作区

## 目录速查

```
📁 产品文档/                ← 你（产品经理）日常使用
   📁 01-产品规划/          ← 产品愿景、Roadmap、业务框架、优先级
   📁 02-需求设计/          ← 一个功能一篇 PRD/设计方案
   📁 03-讨论决策/          ← 与 AI 重要讨论的结论沉淀
   📁 04-工作日志/          ← 周报、复盘、进展追踪

📁 原型预览/                ← 静态 HTML 原型（双击或 dev-server 打开）

📁 docs/                    ← 给 AI 看的"项目宪法"（架构 / 命名 / 加新页流程）

📁 styles/  scripts/  partials/   ← 主原型代码（按 AI 友好结构拆分）

📄 创建需求-上传页面.html   ← 主原型入口
📄 dev-server.py            ← 本地开发服务器
```

## 怎么用

### 启动主原型
```bash
cd /Users/mason/Desktop/Cursor/
python3 dev-server.py
# 浏览器打开 http://localhost:8765/创建需求-上传页面.html
```

### 写新需求设计
1. 在 `产品文档/02-需求设计/` 新建一个 .md 文件
2. 文件名格式：`{模块名}-{日期}.md`，例：`AI对话页-2026-04.md`
3. 模板见 `产品文档/02-需求设计/_模板.md`

### 让 AI 改代码
跟任何 AI（包括 Cursor 里的 AI）说：
> "请先读 `docs/ARCHITECTURE.md` 和 `docs/CONVENTIONS.md`，然后……"

这样 AI 就不会写出和现有结构冲突的代码。

## 重要约定

- **不删源文件**：所有重构、迁移都先备份（如 `创建需求-上传页面.html.backup`）
- **AI 友好命名**：代码部分用英文，PM 文档用中文 —— 互不干扰
- **三件套规则**：每加一个页面 = 1 HTML partial + 1 CSS + 1 JS（详见 `docs/ADD-NEW-PAGE.md`）
