# 架构重构方案 - 主原型从单文件拆分为模块化结构

## 背景

主原型 `创建需求-上传页面.html` 已经达到 **9883 行**：
- CSS 4900 行（49.6%）
- JS 3914 行（39.6%）
- HTML 模版 1070 行（10.8%）

继续在单文件里堆代码会有 4 个明显问题：

1. CSS 互相干扰（多次返工都因为这个）
2. AI 改一处要扫 9000 行才能定位
3. 数据写死在 JS 里，未来接 API 时要满文件搜索
4. 没有命名空间，全局函数已 100+，撞名风险高

## 方案对比

| 方案 | 描述 | 工具链 | 迁移成本 | 适用阶段 |
|---|---|---|---|---|
| **A. 纯文件拆分** | 多 .html/.css/.js + `<link>`/`<script>` 引入 | 零 | 1 周 | ✅ 当前 |
| B. ES Modules + Web Components | import/export + Shadow DOM | 零 | 2 周 | 中期 |
| C. Vite + Vue/React | 完整 SPA 框架 | npm + Vite | 1-2 月 | 团队化时 |

## 决策

✅ **采纳方案 A**

理由：
- 单人开发，不引入构建复杂度
- 80% 现有代码可保留（不是推翻重写）
- ES Modules 是方案 A 的超集，未来可无痛升级到 B

## 6 步迁移路径

| Step | 内容 | 时长 | 验证方式 |
|---|---|---|---|
| 0 | 备份 + 建目录 + 写 AI 宪法 | 10 min | 目录创建好 |
| 1 | CSS 拆分（4900 行 → ~15 个 .css） | 30 min | 视觉无变化 |
| 2 | 数据层抽离（mock 数据 → `scripts/data/`） | 20 min | 数据正常显示 |
| 3 | core 基础设施抽离（icons/toast/router/livereload） | 20 min | 视图切换正常 |
| 4 | 页面 JS 模块化（一页一文件） | 60 min | 每页交互正常 |
| 5 | HTML partials 抽离 + 完善文档 | 30 min | 入口减到 ~50 行 |

## 5 条 AI 协作铁律

1. **命名前缀绑定文件**：`.cf-*` 只在 `copy-list.css`，`copyXxx()` 只在 `copy-list.js`
2. **数据/视图/逻辑分离**：mock 数据只放 `scripts/data/`
3. **一页 = 三件套**：1 partial + 1 css + 1 js
4. **禁止跨页 import**：要共用就抽到 `core/`
5. **入口只有 link/script**：业务代码不写在 `创建需求-上传页面.html`
