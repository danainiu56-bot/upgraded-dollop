# 产品文档

## 目录用途

| 目录 | 放什么 | 文件命名建议 |
|---|---|---|
| **01-产品规划** | 产品愿景、Roadmap、业务框架、需求优先级 —— 长期文档 | 直接用主题命名，例：`营销系统业务框架图.md` |
| **02-需求设计** | 单个功能的 PRD / 交互设计 / 验收标准 | `{模块}-{日期}.md`，例：`AI对话页-2026-04.md` |
| **03-讨论决策** | 与 AI 的重要讨论沉淀、架构决策、方案对比 | `{日期}-{主题}.md`，例：`2026-04-24-架构重构方案.md` |
| **04-工作日志** | 周报 / 复盘 / 当日进展 | `{日期}-{类型}.md`，例：`2026-04-24-日志.md` |

## 写作建议

- 一篇文档解决一件事，**不要 1 篇 5000 字**
- 标题用 H2/H3，便于 AI 抽取上下文
- 决策类文档结尾留「**结论 / 行动项**」单独段落
- 文末加日期 + 状态：`📅 2026-04-24 · ✅ 已确认 / ⏳ 进行中 / ❌ 已废弃`

## 给 AI 用的小技巧

跟 AI 说：
> "参考 `产品文档/03-讨论决策/2026-04-24-架构重构方案.md`，按方案 A 继续推进 Step 3"

AI 会精准带上上下文，不需要你每次重新解释。
